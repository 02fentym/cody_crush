YAML Test Case Format

You can upload a .yaml or .yml file containing test cases for code questions that accept standard input.

Example format:

test_style: stdin
    test_cases:
      - input: "2\n3\n"
        output: "5\n"
      - input: "10\n20\n"
        output: "30\n"
      - input: "-5\n5\n"
        output: "0\n"
  • test_style must be stdin or function (if supported).
  • Each entry under test_cases should include input and output.
  • Use \n for newlines inside the quoted strings.
  • Make sure your YAML uses spaces, not tabs, and is properly indented.
ZIP Test Case Format

You can upload a .zip file that contains:

  • A YAML file (e.g. meta.yaml) that lists .in / .out file names
  • The actual .in and .out files with test content

Example meta.yaml:

test_style: stdin
    test_cases:
      - input: "1.in"
        output: "1.out"
      - input: "2.in"
        output: "2.out"

The ZIP must include the files 1.in, 1.out, 2.in, and 2.out at the root level. Inline strings will be ignored — inputs and outputs must be stored in separate text files and referenced by name in the YAML.
